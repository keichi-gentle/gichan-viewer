[2026-03-30] 세션 1: 프로젝트 기획 및 초기 설정

■ 프로젝트 배경
- 기존 LifeRecord(기찬다이어리) WPF 앱은 PC/랩탑에서만 사용 가능
- 스마트폰/태블릿에서 기록 조회 및 리포트 분석을 위한 동반앱 필요
- 입력은 WPF에서만, 모바일은 조회 전용

■ 기술 검토
- 멀티 디바이스 접근을 위한 선택지 분석:
  · Google Sheets 연동 (A/B/C안): WPF 자체가 Windows 전용이라 모바일 접근 불가 → 기각
  · 웹앱 신규 개발: 과도한 개발 범위 → 보류
  · .NET MAUI: iOS 빌드에 Mac 필요, 생태계 미성숙 → 기각
  · PWA: 앱스토어 불필요, 서버 비용 0원, 브라우저에서 앱처럼 동작 → 채택

■ 설계 결정사항
- 화면 구성: 현황(대시보드) + 조회(리스트) + 리포트(차트) + 설정(Import/페이지건수)
- 디자인: WPF 색상 팔레트 유지 + 모바일 레이아웃 최적화 (C안 채택)
- 차트: Chart.js (가볍고 모바일 터치 지원)
- Excel 가져오기: 파일 선택기 방식 (Google Drive API 연동 없이 단순하게)
- 데이터 캐시: IndexedDB (오프라인 지원)
