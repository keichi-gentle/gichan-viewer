기찬뷰어 (Gichan Viewer) - 작업 메모 기록

[2026-03-30] 세션 1: 프로젝트 기획 및 초기 설정
================================================================================

■ 프로젝트 배경
- 기존 LifeRecord(기찬다이어리) WPF 앱은 PC/랩탑에서만 사용 가능
- 스마트폰/태블릿에서 기록 조회 및 리포트 분석을 위한 동반앱 필요
- 입력은 WPF에서만, 모바일은 조회 전용

■ 기술 검토
- 멀티 디바이스 접근을 위한 선택지 분석:
  · Google Sheets 연동 (A/B/C안): WPF 자체가 Windows 전용이라 모바일 접근 불가 → 기각
  · 웹앱 신규 개발: 과도한 개발 범위 → 보류
  · .NET MAUI: iOS 빌드에 Mac 필요, 생태계 미성숙 → 기각
  · PWA: 앱스토어 불필요, 서버 비용 0원, 브라우저에서 앱처럼 동작 → 채택

■ 설계 결정사항
- 화면 구성: 현황(대시보드) + 조회(리스트) + 리포트(차트) + 설정(Import/페이지건수)
- 디자인: WPF 색상 팔레트 유지 + 모바일 레이아웃 최적화 (C안 채택)
- 차트: Chart.js (가볍고 모바일 터치 지원)
- Excel 가져오기: 파일 선택기 방식 (Google Drive API 연동 없이 단순하게)
- 데이터 캐시: IndexedDB (오프라인 지원)

■ 프로젝트 구조 생성
- ExtendPrj/ 폴더 생성 (Gi_chan 하위)
- src/gichan-viewer/ — PWA 소스 코드
- docs/ — 요구사항, 설계 문서, 작업 기록
- design/ — GUI 디자인 자료
- git init 완료 (로컬 저장소)

■ PWA 구현 완료
- index.html: SPA 구조, 하단 탭바 4탭 (현황/조회/리포트/설정)
- css/style.css: WPF 동일 색상팔레트 (Set 10 로우라이트), Dark/Light 테마, 모바일 레이아웃
- js/app.js: 앱 초기화, 탭 전환, 테마 관리
- js/calc.js: WPF CalculationService → JS 포팅 (수유/배변/위생 계산 로직)
- js/excel-parser.js: SheetJS 기반 Excel A~Q 컬럼 파싱 (레거시 카테고리 호환)
- js/storage.js: IndexedDB 데이터 캐시 + LocalStorage 설정 저장
- js/dashboard.js: 현황탭 6개 카드 (수유/배변/위생/신체/건강/기타)
- js/browse.js: 조회탭 필터/검색/페이지네이션/카테고리별 색상
- js/report.js: 리포트탭 Chart.js 6종 차트 + 요약 카드 5개 (wrap 레이아웃)
- js/settings.js: Excel Import/페이지건수/아기이름/테마 전환
- manifest.json + sw.js: PWA 오프라인 캐시
- icons: 192/512 PNG 아이콘 (G자 모양 틸 색상)
- 외부 라이브러리: SheetJS 273KB + Chart.js 201KB
